F.A.Q
What is VisualOps?

VisualOps is a visual DevOps automation tool. It automates the tasks of resource provisioning, code deployment, configuration managent, to ensure your cloud application always runs in the desired state.

VisualOps is created with a goal of both "easy to understand" and "total control". We believe these two factors are important for any size of users to adopt a technology, whether this means individual project, startup, ISV, enterprise and everywhere in between. VisualOps is appropriate for managing small setups with a handful of instances as well as complicated environments with many thousands.

How VisualOps works?
  • drag-n-drop components to build your AWS infrastructure;
  • clicks instances to setup the software configuration (package, file, code, etc.);
  • single click to deploy, within minutes the app is ready to use!
  • VisualOps continuously ensures the app always runs in the defined states, by repeatedly verifying the instance configuration and fixing any drift!
What are the use cases?

You can use VisualOps in the following scenarios:

  • Template-based provisioning: fast and repeatable Dev/QA, Demo environment setup
  • Continuous Deployment: launch your stack into running app, VisualOps deploys your code and upgrades it once there is a newer version
  • App Ensure: design, launch, relax; VisualOps will ensure your app is always up & running, in the state you defined
What is the main difference between VisualOps and other DevOps tools?

What is the main difference between VisualOps and CloudFormation?

In terms of templating your AWS infrastructure with fast and repeatable deployment, VisualOps shares the same goal as CloudFormation. In this regard, despite the visual part, you can use VisualOps as you use CloudFormation. However, the difference is that VisualOps doesn't only manage the lifecyle of the infrastructure (provision, update, destroy, etc.), it also continuously manages the application running on top of it. If some state drift occurs, VisualOps will spot the issue, fix it and bring the system back to the desired state.

What is the main difference between VisualOps and OpsWorks?

"AWS OpsWorks is an application management service that makes it easy for DevOps users to model and manage the entire application" [1]. OpsWorks is tightly coupled with Chef, which is a significant advantage/disadvantage at the same time. Compared with OpsWorks/Chef's bottom-up approach, VisualOps takes a different route: focus on the infrastructure level first, use the visual IDE to design the architecture; then drill down into the servers to configure and connect them. We believe this is the right approach of the DevOps toolchain: system architecture first, then server details.

How about the security?

We continue to put enormous efforts to harden our security.

  • Everything is encrypted We encrypt all data at every step of the way: all data (both your browser and the agent) transmitted via 256-bit Comodo SSL, and stored encrypted in our database. Even in the log files, your sensitive information, such as username, email address, are logged in the encrypted format.

  • Strict security process Staffs are not allowed to decrypt any user data without the explicit permission from the user (mostly for support case), and they do not have the ability to decrypt any data, unless a permission from our security officer.

  • Verified by AWS We are an official technical partner for Amazon Web Services. Also, we are hosted on AWS as well, which achieved ISO 27001 certification, PCI DSS Level 1 compliance, and SAS70 Type II.
